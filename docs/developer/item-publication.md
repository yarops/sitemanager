# Публикация сайтов

Объект `Item` проходит два независимых этапа: подготовку к публикации и мониторинг доступности. Не смешивайте их статусы.

## Статус публикации

Поле `Item::publish_status` определяет, готов ли сайт к использованию:

| Значение | Значение для редактора | Поведение системы |
| --- | --- | --- |
| `draft` | Сайт создан или находится в подготовке | Видим редактору во фронтенд-списках, но не участвует в мониторинге. |
| `publish` | Сайт выложен | Видим в обычных списках и может участвовать в мониторинге. |

Новый `Item` всегда создаётся со значением `draft`. Редактор заполняет данные сайта и явно переводит его в `publish` после фактического размещения.

### Дата публикации

`publish_date` — это дата первой публикации, а не дата создания записи:

- для черновика значение должно быть `NULL`;
- при первом переходе из `draft` в `publish` модель выставляет текущее время, если редактор не указал дату вручную;
- повторное редактирование опубликованного сайта не меняет `publish_date`; для этого используется `updated_at`.

## Видимость на фронтенде

Авторизованный редактор должен видеть и `draft`, и `publish` сайты в следующих местах:

- общий список и поиск сайтов;
- список сайтов сервера;
- список сайтов шаблона;
- карточка сайта.

Черновик на карточке помечается бейджем «Черновик». Неавторизованным пользователям в публичных списках серверов и шаблонов показываются только опубликованные сайты.

## Связь с мониторингом

Статус публикации не заменяет `check_enabled`:

- `publish_status` отвечает за готовность сайта к публикации;
- `check_enabled` отвечает за включение автоматических проверок уже опубликованного сайта.

Планировщики, ручные массовые проверки, дашборд мониторинга и ежедневные отчёты должны выбирать только сайты со всеми условиями:

```php
[
    'publish_status' => Item::STATUS_PUBLISH,
    'check_enabled' => 1,
    'is_archived' => 0,
]
```

Черновик не должен попадать в очередь проверок и не должен генерировать уведомления, даже если у него случайно включён `check_enabled`.

Архивированный сайт также исключается из списков и мониторинга независимо от статуса публикации.
